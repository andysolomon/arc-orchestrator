# Cursor Opus Review

Use this skill when the task needs taste/design judgment more than implementation throughput.

## Use Opus 4.8 For

- UI/UX polish and confusing states;
- accessibility and user-facing copy;
- API ergonomics and naming;
- component composition and architecture boundaries;
- developer-experience docs;
- prompt or skill wording;
- second-opinion design critique after Composer or Codex implementation.

## Avoid Opus For

- bulk implementation;
- mechanical migrations;
- large repo scans;
- straightforward test additions;
- generic CI/log summarization;
- basic correctness/security review better handled by Codex check.

## Output

Return a read-only critique with verdict, top findings, evidence, suggested improvements, tradeoffs, and whether Composer or Codex should perform follow-up implementation.
